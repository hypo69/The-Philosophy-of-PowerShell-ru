פילוסופיית PowerShell.

חלק 0.

מה היה לפני PowerShell? בשנת 1981 יצא MS-DOS 1.0 עם מפרש הפקודות `COMMAND.COM`. לאוטומציה של משימות שימשו קובצי אצווה (`.bat`) — קובצי טקסט פשוטים עם רצף פקודות קונסולה. אסקטיזם מדהים של שורת הפקודה על רקע מערכות תואמות POSIX שבהן כבר משנת 1979 התקיימה מעטפת בורן (`sh`).

📅 מצב שוק המעטפות בזמן יציאת **MS-DOS 1.0** (אוגוסט 1981)

להלן טבלה מסכמת של מערכות ההפעלה הפופולריות באותה תקופה ותמיכתן במעטפות (`sh`, `csh` ועוד):

מערכת הפעלה תמיכה במעטפות (`sh`, `csh`, ועוד) הערה
**UNIX Version 7 (V7)** `sh` UNIX קלאסי אחרון של Bell Labs, נפוץ מאוד
**UNIX/32V** `sh`, `csh` גרסת UNIX לארכיטקטורת VAX
**4BSD / 3BSD** `sh`, `csh` ענף אוניברסיטאי של UNIX מברקלי
**UNIX System III** `sh` גרסה מסחרית ראשונה מ-AT&T, קודמת ל-System V
**Xenix (מ-Microsoft)** `sh` גרסת UNIX מורשית, נמכרה על ידי Microsoft משנת 1980
**IDRIS** `sh` מערכת הפעלה דמוית UNIX עבור PDP-11 ו-Intel
**Coherent (Mark Williams)** `sh` (דומה) אלטרנטיבה זולה ל-UNIX עבור מחשבים אישיים
**CP/M (Digital Research)** ❌ (אין `sh`, רק CLI פשוט ביותר) לא UNIX, מערכת ההפעלה הפופולרית ביותר למחשבי 8 סיביות
**MS-DOS 1.0** ❌ (רק `COMMAND.COM`) מעטפת פקודה מינימלית, ללא סקריפטים או צינורות

💡 מהם `sh`, `csh`

  • `sh` — **מעטפת בורן** (Bourne Shell), מפרש הסקריפטים העיקרי של UNIX משנת 1977.
  • `csh` — **מעטפת C** (C Shell), מעטפת משופרת עם תחביר דמוי C ונוחות לעבודה אינטראקטיבית.
  • מעטפות אלו **תמכו בהפניות, צינורות, משתנים, פונקציות ותנאים** — כל מה שהפך את UNIX לכלי אוטומציה רב עוצמה.

מיקרוסופט התמקדה ב**מחשבי IBM PC זולים של 16 סיביות**, שהיו בעלי **מעט זיכרון** (בדרך כלל 64–256 קילובייט), חסרו ריבוי משימות, ונועדו ל**שימוש ביתי ומשרדי**, לא לשרתים. UNIX היה יקר, דרש ארכיטקטורה מורכבת ומומחיות, בעוד שרואי חשבון ומהנדסים, לא מנהלי מערכת, נזקקו למערכת הפעלה מהירה ופשוטה.

ממשק DOS, במקום ה-`sh` המורכב, הציג קובץ command.com יחיד עם סט דל של פקודות פנימיות (dir, copy, del וכו') ללא פונקציות, לולאות או מודולים.

היו גם פקודות חיצוניות — קבצים ניתנים להרצה נפרדים (.exe או .com). דוגמאות: FORMAT.COM, XCOPY.EXE, CHKDSK.EXE, EDIT.COM. סקריפטים לביצוע נכתבו בקובץ טקסט עם סיומת .bat (batch file).

דוגמאות לקובצי תצורה:

  • AUTOEXEC.BAT
:: ------------------------------------------------------------------------------
:: AUTOEXEC.BAT — Automatic configuration and launch of Windows 3.11
:: Author: hypo69
:: Year: approximately 1993
:: Purpose: Performs DOS environment initialization, loads network drivers, and launches Windows 3.11
:: ------------------------------------------------------------------------------
@ECHO OFF

:: Command prompt setting
PROMPT $p$g

:: Environment variables setting
SET TEMP=C:\TEMP
PATH=C:\DOS;C:\WINDOWS

:: Loading drivers and utilities into upper memory
LH C:\DOS\SMARTDRV.EXE       :: Disk cache
LH C:\DOS\MOUSE.COM          :: Mouse driver

:: Loading network services (relevant for Windows for Workgroups 3.11)
IF EXIST C:\NET\NET.EXE LH C:\NET\NET START

:: Automatic Windows launch
WIN
  • CONFIG.SYS

במיקרוסופט, במקביל ל-DOS, כמעט מיד החלו לפתח ליבה חדשה לחלוטין.

ליבת **Windows NT** (New Technology) הופיעה לראשונה עם שחרור מערכת ההפעלה:

**Windows NT 3.1 — 27 ביולי 1993**

  • **הפיתוח החל**: ב**1988** תחת הנהגתו של **דייב קאטלר** (Dave Cutler) (מהנדס לשעבר ב-DEC, יוצר VMS) במטרה ליצור מערכת הפעלה חדשה לחלוטין, מאובטחת, ניידת ורב-משימתית, שאינה תואמת ל-MS-DOS ברמת הליבה.
  • **NT 3.1** — נקראה כך כדי להדגיש תאימות ל**Windows 3.1** ברמת הממשק, אך הייתה בעלת **ארכיטקטורה חדשה לחלוטין**.

🧠 מה הביאה ליבת NT:

תכונה תיאור
**ארכיטקטורת 32 סיביות** בניגוד ל-MS-DOS ו-Windows 3.x, שהיו 16 סיביות.
**ריבוי משימות** ריבוי משימות מקדים אמיתי.
**זיכרון מוגן** תוכניות לא יכלו להשחית את הזיכרון זו של זו.
**מודולריות** ארכיטקטורת ליבה רב-שכבתית: HAL, Executive, Kernel, דרייברים.
**תמיכה בריבוי פלטפורמות** NT 3.1 פעלה על x86, MIPS ו-Alpha.
**תאימות ל-POSIX** NT סופקה עם **תת-מערכת POSIX**, מאושרת ל-POSIX.1.

מיקרוסופט השתמשה בגישות, אסטרטגיות וכלים שונים לחלוטין לניהול. כל זה היה **מפורק**, לעיתים קרובות מונחה GUI ולא תמיד ניתן לאוטומציה.

📌 רשימת כלים מסוימים:
כלי מטרה

🛠 דוגמאות לאוטומציה:
  • קובצי VBScript (*.vbs) לניהול משתמשים, רשתות, מדפסות ושירותים.

  • סקריפטים של `.cmd` עם קריאות ל-`net`, `sc`, `reg`, `wmic` וכו'.

    ועכשיו — הדבר החשוב ביותר!

    היתרון העיקרי של PowerShell בהשוואה למעטפות פקודה קלאסיות הוא שהוא עובד עם *אובייקטים*, ולא עם טקסט. כאשר אתה מפעיל פקודה, היא מחזירה לך לא רק טקסט, אלא אובייקט מובנה (או אוסף של אובייקטים), שיש לו מאפיינים (Properties) ושיטות (Methods) מוגדרים בבירור.

    ראה כיצד PowerShell עולה על מעטפות קלאסיות בזכות **עבודה עם אובייקטים**.

    📁 איך זה היה: `dir` וניתוח ידני

    ב-**CMD** (גם ב-`COMMAND.COM` הישן וגם ב-`cmd.exe`), הפקודה `dir` מחזירה את התוצאה כטקסט רגיל. דוגמת פלט:

    24.07.2025  21:15         1 428  my_script.js
25.07.2025  08:01         3 980  report.html

    נניח שאתה רוצה לחלץ את **שם הקובץ** ו**גודלו** של כל קובץ. תצטרך לנתח את השורות ידנית:

    for /f "tokens=5,6" %a in ('dir ^| findstr /R "[0-9][0-9].[0-9][0-9].[0-9][0-9][0-9][0-9]"') do @echo %a %b
    • זה קשה נורא לקריאה, תלוי ב-locale, בפורמט התאריך, בגופן. ונשבר עם רווחים בשמות.

    ✅ PowerShell: אובייקטים במקום טקסט

    ✔ דוגמה פשוטה וקריאה:

    Get-ChildItem | Select-Object Name, Length

    **תוצאה:**

    Name          Length
----          ------
my_script.js   1428
report.html    3980
    • `Get-ChildItem` מחזיר **מערך של אובייקטי קבצים/תיקיות**
    • `Select-Object` מאפשר לך לקבל בקלות את ה**מאפיינים** הדרושים

    🔍 מה באמת מחזיר `Get-ChildItem`?

    $item = Get-ChildItem -Path .\my_script.js
$item | Get-Member

    **תוצאה:**

    TypeName: System.IO.FileInfo

Name         MemberType     Definition
----         ---------      ----------
Length       Property       long Length {get;}
Name         Property       string Name {get;}
CreationTime Property       datetime CreationTime {get;set;}
Delete       Method         void Delete()
...

    PowerShell מחזיר **אובייקטים מסוג `System.IO.FileInfo`**, שיש להם:

    • 🧱 מאפיינים (`Name`, `Length`, `CreationTime`, `Extension`, …)
    • 🛠 שיטות (`Delete()`, `CopyTo()`, `MoveTo()` וכו')

    אתה עובד **עם אובייקטים מלאים**, לא עם מחרוזות.

    תחביר "פועל-שם עצם":

    PowerShell משתמש ב**תחביר פקודות קפדני והגיוני**: "פועל-שם עצם" (Verb-Noun)

    פועל מה הוא עושה
    `Get-` קבל
    `Set-` הגדר
    `New-` צור
    `Remove-` הסר
    `Start-` התחל
    `Stop-` עצור
    שם עצם על מה הוא עובד
    `Process` תהליך
    `Service` שירות
    `Item` קובץ/תיקיה
    `EventLog` יומני אירועים
    `Computer` מחשב

    🔄 דוגמאות:

    מה צריך לעשות פקודה
    קבל תהליכים `Get-Process`
    עצור שירות `Stop-Service`
    צור קובץ חדש `New-Item`
    קבל תוכן תיקיה `Get-ChildItem`
    מחק קובץ `Remove-Item`

    ⬅️ גם אם אתה **לא יודע את הפקודה המדויקת**, אתה יכול **לנחש** אותה לפי משמעותה — וכמעט תמיד תצליח.

    ה-cmdlet `Get-Help` הוא העוזר הראשי שלך.

    1. **קבל עזרה על העזרה עצמה:** 
      Get-Help Get-Help
    2. **קבל עזרה בסיסית על הפקודה לעבודה עם תהליכים:** 
      Get-Help Get-Process
    3. **ראה דוגמאות לשימוש בפקודה זו:** 
      Get-Help Get-Process -Examples

      זהו פרמטר שימושי להפליא שלעיתים קרובות מספק פתרונות מוכנים למשימות שלך.

    4. **קבל את המידע המפורט ביותר על הפקודה:** 
      Get-Help Get-Process -Full
